# Dominio e GitHub Pages

Questa procedura va eseguita soltanto dopo la clearance del nome.

## Architettura consigliata

- dominio: semplice indirizzo pubblico;
- hosting: GitHub Pages;
- download: GitHub Releases;
- sorgente: repository GitHub;
- documenti aziendali: mai caricati sul sito;
- analytics e cookie: assenti per impostazione predefinita.

## Passaggi

1. acquistare il dominio presso un registrar scelto dall'utente;
2. proteggere l'account con MFA e conservare i codici di recupero;
3. impostare il rinnovo automatico;
4. aggiungere il dominio personalizzato nelle impostazioni di GitHub Pages;
5. configurare i record DNS indicati da GitHub;
6. abilitare HTTPS;
7. verificare sia il dominio principale sia `www`;
8. pubblicare un indirizzo email che non prometta assistenza garantita;
9. controllare annualmente dominio, DNS e recapiti.

## Privacy

La pagina statica non deve includere moduli che raccolgono documenti, tracker pubblicitari
o script di profilazione. Un eventuale modulo contatti va aggiunto soltanto dopo avere
definito titolare, informativa, conservazione e gestione delle richieste.
